# ADR-003: 缓存层选型

> 状态：**已决策** | 更新日期：2025-01-15 | 参与人：API 网关团队

---

## 1. 背景

API 网关需要引入缓存层以降低数据库压力。

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| P99 延迟 | **210 ms** | **< 50 ms** |
| 数据库 QPS | 峰值 12k | 降低 70%+ |

---

## 2. 候选方案

| 维度 | 方案 A：Redis Cluster | 方案 B：本地内存缓存 | 方案 C：Cloudflare KV |
|------|-----------------------|----------------------|----------------------|
| **成熟度** | 高，广泛应用于业界 | 高，无外部依赖 | 中，边缘计算场景 |
| **运维成本** | 中等（需维护集群） | **零** | 低（托管服务） |
| **数据共享** | 支持多实例共享 | 不支持跨实例 | 全球边缘共享 |
| **P99 延迟** | **~45 ms** | ~5 ms（本地访问） | 50~200 ms（波动大） |
| **命中率** | 高（统一缓存池） | 低（各实例独立） | 中（边缘分布） |
| **团队经验** | 已有运维经验 | 经验丰富 | 无 |
| **社区生态** | 成熟 | N/A | 快速演进中 |

---

## 3. 决策

**选择方案 A：Redis Cluster**

### 核心理由

1. **性能达标** -- P99 延迟可降至 **45 ms**，满足 50 ms 的目标
2. **团队优势** -- 团队已有 Redis 运维经验，学习成本低
3. **生态成熟** -- 客户端库、监控工具、最佳实践丰富，踩坑风险小
4. **共享缓存** -- 支持多网关实例共享同一缓存池，缓存效率最大化

### 未选方案说明

- **方案 B（本地内存缓存）** 虽延迟最低且零运维，但不支持多实例共享，命中率不足，无法有效降低数据库压力
- **方案 C（Cloudflare KV）** 延迟波动大，P99 不稳定，且团队缺乏相关运维经验，不适合延迟敏感场景
